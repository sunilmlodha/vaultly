# Vaultly Regression Testing Pack

End-to-end regression suite covering every user journey in Vaultly.
Built with [Playwright](https://playwright.dev/).

---

## Structure

```

│
├── api/                       # Vitest API route tests (mocked db + auth)
│   ├── assets.test.ts         # GET/POST/PUT/DELETE /api/assets
│   ├── goals.test.ts          # GET/POST/PUT/DELETE /api/goals
│   ├── renewals.test.ts       # GET/POST/PUT/DELETE /api/renewals
│   └── account.test.ts        # GET/PATCH/DELETE /api/account
│
├── components/                # Vitest component tests (Testing Library)
│   └── ui.test.tsx            # Badge, Button, Card, Input
│
└── regression/                # Playwright E2E regression pack ← this folder
    ├── README.md              # This file
    ├── setup/
    │   └── auth.setup.ts      # Logs in once, saves session to fixture
    ├── fixtures/
    │   ├── test-data.ts       # Shared test data + helpers
    │   ├── auth-state.json    # Generated by auth setup (git-ignored)
    │   └── test-doc.pdf       # Generated by documents journey
    ├── journeys/
    │   ├── 01-dashboard.spec.ts     # Dashboard loads, stat cards, charts
    │   ├── 02-assets.spec.ts        # Add → edit → delete assets
    │   ├── 03-liabilities.spec.ts   # Add → edit → delete liabilities
    │   ├── 04-goals.spec.ts         # Add → AI coach → edit → delete goals
    │   ├── 05-renewals.spec.ts      # Add → negotiate → edit → delete renewals
    │   ├── 06-documents.spec.ts     # Upload → analyse → delete documents
    │   ├── 07-settings.spec.ts      # Currency, language switch, delete account modal
    │   ├── 08-connections.spec.ts   # Open Banking UI, connect, disconnect
    │   ├── 09-spending-forecast.spec.ts  # Spending analytics, cashflow forecast
    │   ├── 10-family.spec.ts        # Invite member, view household

```

## Quick Start

### Prerequisites

```bash
# Install dependencies (if not already done)
npm install

# Install Playwright browsers
npx playwright install chromium
```

### Running the full regression pack

```bash
# Against local dev server (auto-starts)
npm run test:regression

# Against a specific URL (e.g. staging or production)
BASE_URL=https://vaultly-mu.vercel.app npm run test:regression

# Interactive UI mode (explore tests, see traces)
npm run test:regression:ui

# Headed mode (watch the browser)
npm run test:regression:headed
```

### Running unit + API + component tests only

```bash
npm run test               # Run all Vitest tests once
npm run test:watch         # Watch mode during development
npm run test:unit          # Unit tests only
npm run test:api           # API tests only
npm run test:components    # Component tests only
npm run test:coverage      # Generate coverage report
```

### Running everything

```bash
npm run test:all           # Vitest + Playwright
```

## CI / Vercel Integration

Add to your CI pipeline (GitHub Actions example):

```yaml
- name: Install dependencies
  run: npm ci

- name: Install Playwright browsers
  run: npx playwright install --with-deps chromium

- name: Build
  run: npm run build

- name: Unit & API tests
  run: npm run test

- name: Regression tests
  run: npm run test:regression
  env:
    BASE_URL: ${{ secrets.STAGING_URL }}
    TEST_EMAIL: ${{ secrets.TEST_EMAIL }}
    TEST_PASSWORD: ${{ secrets.TEST_PASSWORD }}
```

---

## Environment Variables

| Variable | Default | Description |
|---|---|---|
| `BASE_URL` | `http://localhost:3000` | App URL to test against |
| `TEST_EMAIL` | `test@vaultly.app` | Email for the test account |
| `TEST_PASSWORD` | `TestPassword123!` | Password for the test account |
| `CI` | unset | Set to `true` in CI — enables retries, disables dev server |

---

## Test Account Requirements

The regression tests require a real (or seeded) user account with:
- Valid email + password credentials
- At least 1 household record in the database

For local testing, create an account via `/signup` first, then set `TEST_EMAIL` and `TEST_PASSWORD`.

For CI, provision a dedicated test account and add its credentials as secrets.

## Adding Tests for New Features

1. Create `tests/regression/journeys/NN-feature-name.spec.ts`
2. Follow the existing journey pattern:
   - `test.describe('Feature Name', () => { ... })`
   - `test.beforeEach` navigates to the page
   - Each `test()` covers a specific action or assertion
3. Add unit tests in `tests/unit/` for any new lib functions
4. Add API tests in `tests/api/` for any new API routes
5. Run `npm run test:all` before merging
